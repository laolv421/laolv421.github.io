JWT（JSON Web Token）格式

Posted on 2018-10-08 |

JWT格式

Bearer token在OIDC中定义的格式为JWT（发音为[jot]），JWT，JSON Web Token是一个简洁的表达格式，用于JSON Web Signature（JWS）的载荷或者JSON Web Encryption（JWE）加密的明文；JWT能够提供签名保护和消息完整的保护，通常使用JWS Compact Serialization或者JWE Compact Serialization表达。

JWT结构

JWT是由一系列URL-safe，由被”.”分割的parts组成，每个part是经过base64url编码的。parts的个数是由JWS或者JWE决定的，常见的是三个parts。例如下面的例子是一个JOSE header（JWS{header、payload、signature}定义的第一个部分），header主要描述了jwt的基本信息，如类型、签名所用算法等：

JWT的存储

如果对于纯web来说，存储方式无非两种，HTML5 Web Storage和Cookie。

JWT安全性分析

对于Web Storage存储方式来说，任何JS脚本都可以访问（same domain），也就是说它易受XSS攻击；Cookie存储通过设置HttpOnly可以防止JS脚本访问cookie，同时可以设置Secure标签来保证仅在HTTPS环境下传输此cookie，但是使用了cookie就打破了RESTful best practice的规则。

而且cookie易受CSRF的攻击，通过盗用cookie身份来访问有害网站去执行一些危险操作。可以通过同步CSRF token 来阻止CSRF的攻击，所有的现代web框架目前都支持这种操作。原理是在HTTP请求中以参数的形式加入一个随机产生的 token，并在服务器端建立一个拦截器来验证这个 token，如果请求中没有 token 或者 token 内容不正确，则认为可能是 CSRF 攻击而拒绝该请求。此时载荷可以如下：
