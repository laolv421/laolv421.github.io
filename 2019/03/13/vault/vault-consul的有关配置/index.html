vault server配置

上面提到vault server部署在consul client的那台机器上，按照上面的方法，copy证书到vault的配置目录vault.d下面，产生consul server的证书和私钥，将它们命名为vault.pem和vault-key.pem，然后新建vault.hcl

lister配置为支持https访问，并开放8200端口；storage存储配置为consul，其中的token表示是vault对consul可以进行哪些操作，可以参考文档，我们为了方便，使用management token；并配置地址端口为8501和https scheme，保证和consul client的访问全部在https下进行。然后运行vault

配置secret engines

接下来配置vault的consul secret engines，动态产生consul api token访问consul，这里可以参考文档。

  1. 激活consul secret engines

  2. 配置vault和consul的连接和认证，consul/config/access配置Consul的访问信息。使用此访问信息，以便Vault可以与Consul通信并生成Consul令牌，查看文档。

  3. 配置一个Vault role的名称，将它映射到Consul ACL策略的角色。角色role的名称我们定为access-consul-secrets-role，policy我们定义为access-consul-secrets-policy.json

然后把这个policy写入access-consul-secrets-role这个角色

写入成功之后就可以读取consul/creds/access-consul-secrets-role这个角色的动态凭证来访问vault/secrets下的敏感信息了。之后就可以参考我的《vault-consul集群访问动态凭证》。
